# Security Policy

## Supported Versions

anchor is currently in pre-1.0 development. Only the latest published version receives security fixes.

| Version | Supported |
|---------|-----------|
| 0.2.x (latest) | ✅ Yes |
| < 0.2.0 | ❌ No |

## Reporting a Vulnerability

Do **not** open a public GitHub issue for security vulnerabilities.

Send a report to: **security@accelmars.com**

**Response SLA:** We acknowledge all reports within 48 hours.

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact assessment
- Affected versions

## Scope

**In scope:**
- The accelmars/anchor codebase (all crates and binaries in this repository)
- CLI behavior and data handling

**Out of scope:**
- Third-party dependencies (report these upstream)
- Infrastructure or hosting environments
- Issues already publicly disclosed

## PGP

PGP key pending publication. Reports sent to `security@accelmars.com` are received and handled securely.

## Disclosure and Credit

We will coordinate a fix and disclosure timeline with you. If a CVE is warranted, we will request one. We credit all reporters in the release notes unless anonymity is requested.
